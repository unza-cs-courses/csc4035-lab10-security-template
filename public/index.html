Lab 10: Security Audit

Lab Objective: This application intentionally contains security vulnerabilities. Your task is to identify and fix them.

Vulnerable Endpoints (DO NOT USE IN PRODUCTION)

GET /api/vulnerable/greet?name= - XSS vulnerable endpoint

Try: ?name=<script>alert('XSS')</script>

POST /api/vulnerable/comment - Accepts unsanitized input

Secure Endpoints (After Your Fixes)

GET /api/secure/greet?name= - XSS protected endpoint
POST /api/secure/comment - Input validated and sanitized

Security Measures to Implement

  1. Input Validation: Validate all user input
  2. Input Sanitization: Escape HTML special characters
  3. Security Headers: Configure Helmet.js
  4. Rate Limiting: Prevent abuse with rate limits
  5. Error Handling: Don't leak sensitive info in production

Testing

Run npm test to check your security implementations.

Manual testing with curl or Postman is recommended.
